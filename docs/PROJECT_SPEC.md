## 4. 신청자 화면 사양

### 신청 첫 화면

- URL: `/apply`
- 상단 브랜드 클릭 시 항상 `/apply`로 이동합니다.
- 신청 유형은 두 카드로 분리합니다.
  - 홈스테이 신청: 하트 아이콘
  - 자원봉사자 신청: 십자가 아이콘
- 카드 제목은 다음 두 줄 정렬을 유지합니다.
  - `2027 서울 WYD`
  - `홈스테이 신청` 또는 `자원봉사자 신청`

### 홈스테이 신청

- URL: `/apply/homestay`
- 주요 단계:
  - 서약 확인 및 동의
  - 신청자 인적사항
  - 가족 구성
  - 홈스테이 여건
  - 서명 및 완료
- 신청자는 4자리 숫자 PIN을 등록합니다.
- 가족 구성 단계에서는 1번 구성원을 가족대표 신청자 정보로 자동 매핑하고, 2번 구성원 입력란을 기본으로 노출합니다.
- 접수 후 `/check`에서 성명, 연락처, 4자리 PIN으로 조회합니다.
- 주소는 카카오 우편번호 서비스로 검색하고 우편번호, 기본 주소, 상세 주소를 분리 저장합니다.

### 자원봉사자 신청

- URL: `/apply/volunteer`
- 인적사항:
  - 성명, 세례명, 성별, 생년월일, 연락처, 이메일, 구역, 소속 단체, 주소
  - 생년월일 입력 시 만 나이를 자동 계산합니다.
- 주소 입력 시 홈스테이 신청과 동일하게 구역반을 자동 판별하며, 신청자가 필요하면 구역/반을 수동 설정할 수 있습니다.
- 구역반 안내 링크를 통해 현재 편성 기준을 모달로 확인할 수 있습니다.
- 희망 봉사 분야:
  - 순례자 환대 및 안내
  - 행사 운영 지원
  - 환경 및 시설 관리 지원
  - 외국어 지원
  - 의료 지원
  - 어느 분야든 필요에 따라 봉사 가능합니다.
  - 중복 선택 가능
- 외국어 지원 선택 시 지원 언어를 복수 선택할 수 있습니다.
  - 기본 언어: English, Espanol, Francais, Italiano, Portugues, 日本語, 中文
  - 기타 선택 시 직접 입력란을 노출합니다.
- 봉사 가능 기간:
  - 요일: 평일, 주말, 모두 가능
  - 시간: 오전, 오후, 저녁, 종일 가능
  - 요일과 시간을 각각 복수 선택할 수 있습니다.
- 신청일은 자동 입력되며 수정하지 않습니다.
- 서명란은 성명과 세례명을 자동 매핑해 다음 양식으로 표시합니다.
  - `202X년 XX월 XX일`
  - `신청인 성명(세례명) : xxx (세례명) (서명 / 인)`

## 5. 운영자 화면 사양

### 공통

- URL: `/admin`
- 관리자 로그인은 이메일과 비밀번호를 기본으로 사용합니다.
- 가명/마스킹 정보만 보는 일반 운영자 `admin`은 MFA가 필수는 아닙니다.
- 원본 개인정보에 접근 가능한 `privacy_admin`, `super_admin`은 TOTP MFA가 필수입니다.
- MFA 필수 계정이 최초 MFA 미등록 상태이면 로그인 과정에서 `mfa_secret`을 안내하고 OTP 앱 등록 후 6자리 코드를 검증합니다.
- 운영자 회원가입, 비밀번호 변경, 최고 관리자 승인 워크플로우를 제공합니다.
- `brotheroak@gmail.com`, `livelab21@nate.com`은 고정 최고 관리자이며 다른 운영자가 권한을 변경할 수 없습니다.
- 홈스테이와 자원봉사자는 탭으로 구분합니다.
- 통합 검색은 접수번호, 성명, 연락처, 이메일, 주소, 성별·언어·지원분야를 지원합니다.
- `1구역`처럼 번호가 포함된 검색어는 `11구역`을 함께 반환하지 않고 정확한 구역 번호를 비교합니다.
- `남자`/`남성`, `여자`/`여성` 등 업무상 동의어를 정규화합니다.

### 홈스테이 관리

- 접수 상태별 통계와 수용 가능 인원을 보여줍니다.
- 구역, 침대방, 반려동물, 성별, 호스트 연령대 통계를 탭으로 보여줍니다.
- 수용 인원은 최소 2명이며, 활성 신청서의 연락처 또는 이름과 전체 주소가 같으면 중복 등록을 차단합니다.
- 개인정보 관리자는 신청서와 연계 정보를 트랜잭션으로 영구 삭제할 수 있습니다.
- 신청 카드에서 상태를 변경할 수 있습니다.
- 성별 분포를 별도 대시보드 탭으로 제공합니다.

### 봉사 일정 관리

- 운영자는 일정명, 활동 분야, 장소, 시작·종료 시각, 정원, 안내를 등록·수정·삭제합니다.
- 자원봉사자는 본인 접수 인증 후 공개된 일정에 신청하거나 취소합니다.
- 같은 시간대 일정의 중복 신청과 정원 초과는 서버 트랜잭션 안에서 차단합니다.

### 순례자·호스트 관리

- 개인정보 관리자만 순례자의 교구, 지역, 학년, 나이, 성별, 식단, 알레르기, 발열·건강 메모를 조회·수정합니다.
- 승인된 홈스테이 호스트에 순례자를 배정하거나 해제합니다.
- 순례자 번호를 바코드 카드로 출력하고 카메라 또는 수기 번호 입력으로 카드를 조회합니다.
- 식사 제공 시각과 메모를 이력으로 남겨 중복 배식과 특이 식단을 확인합니다.

### FAQ·Q&A

- 공개 `/community`에서 FAQ를 검색하고 Q&A를 등록·조회합니다.
- 질문 등록에는 개인정보 수집·활용 동의가 필요하며 작성자 이름은 공개 목록에서 마스킹합니다.
- 운영자는 FAQ를 등록·삭제하고 질문에 답변하거나 게시물을 삭제합니다.
- 일반 운영자는 개인정보가 마스킹된 값을 봅니다.
- 개인정보 관리자는 원본 상세 조회 및 수정이 가능합니다.
- 순례자 조건 기반 매칭 후보를 조회할 수 있습니다.

### 자원봉사자 관리

- 총 신청, 승인 대기, 외국어 지원 가능, 의료 지원 가능 지표를 표시합니다.
- 소통 언어 분포 차트를 표시합니다.
- 지원 유형 비율 차트를 표시합니다.
- 샘플 데이터 생성 버튼으로 관리자 화면을 테스트할 수 있습니다.
- 신청 카드에서 상태를 변경할 수 있습니다.

## 6. 개인정보 및 보안 사양

- 운영 모드에서 `DATA_ENCRYPTION_KEY`가 없으면 서버가 시작되지 않습니다.
- 주요 개인정보 컬럼은 AES-256-GCM으로 암호화합니다.
- 신청자 PIN은 신청 ID와 조합한 SHA-256 해시로 저장합니다.
- 인증번호 조회용 이메일은 평문 대신 안정 해시를 사용합니다.
- 일반 운영자 응답에는 개인정보를 마스킹합니다.
- 개인정보 관리자 원본 조회, 수정, 감사 로그 다운로드는 `audit_logs`에 기록합니다.
- `/api`는 IP 기준 rate limit을 적용합니다.
- 동시 처리 요청은 `MAX_CONCURRENT_REQUESTS`로 제한합니다.
