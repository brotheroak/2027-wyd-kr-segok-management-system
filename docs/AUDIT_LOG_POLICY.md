# 감사 로그 다운로드 및 보존 정책

## 목적

감사 로그는 운영자 로그인, 신청서 상태 변경, 개인정보 관리자 수정, 샘플 데이터 생성, 로그 다운로드 같은 보안 이벤트를 추적하기 위한 기록입니다.

## 다운로드

- 다운로드 API: `GET /api/admin/audit-logs.csv`
- 권한: 개인정보 관리자만 가능
- 형식: UTF-8 BOM 포함 CSV
- 파일명: `wyd-audit-logs-YYYYMMDD.csv`
- 다운로드 행위 자체도 `privacy_admin_downloaded_audit_logs`로 감사 로그에 남깁니다.

## 포함 항목

| 컬럼 | 설명 |
| --- | --- |
| `id` | 감사 로그 ID |
| `actor` | 수행자 |
| `action` | 수행 행위 |
| `application_id` | 관련 신청 ID |
| `detail` | 부가 정보 JSON |
| `created_at` | 생성 시각 |

## 보존 정책

- 운영 DB 내부 보존: 행사 종료 후 1년
- CSV 백업 보존: 행사 종료 후 3년
- 접근 권한: 개인정보 관리자 및 책임자
- 공유 방식: 암호화 압축 또는 접근 통제된 저장소
- 삭제 방식: 보존 기간 종료 후 백업본 포함 삭제

## 운영 절차

1. 행사 접수 기간에는 주 1회 CSV를 내려받아 암호화 보관합니다.
2. 개인정보 열람/수정 이슈가 발생하면 해당 기간의 감사 로그를 별도 보존합니다.
3. 다운로드한 CSV는 개인 PC에 장기 보관하지 않습니다.
4. 외부 공유가 필요한 경우 개인정보 책임자 승인 후 최소 범위만 공유합니다.

## 주의사항

감사 로그에는 운영자 식별자와 일부 신청 관련 ID가 포함될 수 있으므로 개인정보에 준해 다룹니다.
